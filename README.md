# Telegram bot: daily EiBi stations

Бот каждый день:

1. в `23:55 UTC` заранее обновляет локальную SQLite-базу данными из `eibi.txt`,
2. в `00:00 UTC` читает записи из базы и фильтрует по текущему дню недели (UTC),
3. оставляет только станции, которые в этот день вещают на одном языке,
4. отправляет список в Telegram с группировкой по языку вещания.

Источники:

- Список: http://www.eibispace.de/dx/eibi.txt
- Описание формата: http://www.eibispace.de/dx/README.TXT

## Установка

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## Настройка

1. Скопируйте `.env.example` в `.env`.
2. Заполните:
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`
   - `EIBI_DB_PATH` (опционально, путь к SQLite базе; по умолчанию `data/eibi.sqlite3`)

Как получить `chat_id`:

- Напишите боту в Telegram.
- Откройте:
  `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
- Возьмите значение `chat.id`.

## Запуск

```powershell
python bot.py
```

После запуска бот будет ждать и:
- обновлять базу ежедневно в `23:55 UTC`,
- отправлять отчет ежедневно в `00:00 UTC`.
Команды читают данные из локальной SQLite-базы.
Также можно запустить ручное обновление командой `/refresh`.

Команды в Telegram:

- `/start` — подсказка по командам.
- `/now` — сначала показать кнопки с языками на сегодня, затем после выбора языка прислать станции только по нему.

В интерфейсе языков используются человекочитаемые подписи (например, `R (Russian)`, `E (English)`), а если код неизвестен — показывается оригинальный код из EiBi.

## Важно про формат дней

В `eibi.txt` есть простые и специальные обозначения дней.
В боте поддержаны:

- ежедневные записи (без поля дней),
- числовой формат (`1234567`, где `1=Mo` ... `7=Su`),
- перечень дней (`Mo,Tu,We`),
- диапазон (`We-Mo`),
- первый N-й день недели в месяце (`1.Sa`, `2.Fr`),
- последний день недели в месяце (`Last7`, `LastFr`),
- “чередующиеся” дни (`altFr` — учитывается как пятница),
- формат с ограничением по дате месяца (`MF-15`),
- формат конкретной даты (`15Sep`).

Строгий режим: редкие метки без однозначной календарной логики (`Ram`, `Haj`, `irr`, `test`, `tent`, `alt` и т.д.) не включаются автоматически.
